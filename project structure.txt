# üß† 1. Core Philosophy ‚Äî "Polyglot Persistence + Service Segmentation"

Each part of your system is assigned:

* üóÉÔ∏è **Primary Database** (for reliability and scale)
* ‚ö° **Cache Layer** (for low latency)
* üß© **Language / Framework** (for code implementation)
* üîÅ **Reason** (why it‚Äôs used there)

Everything plugs into **Kafka (or RabbitMQ)** as an event backbone ‚Äî this allows each service to evolve independently.

# üîÅ 4. Data Flow Example (like Instagram + Omegle)

### Example 1: Feed Post

1. User uploads ‚Üí API Gateway ‚Üí Post Service (Node.js)
2. Post Service saves metadata to MongoDB
3. Publishes "new_post" event to Kafka
4. Feed Worker consumes event ‚Üí updates followers' Redis timeline cache
5. Users‚Äô timeline fetch pulls from Redis ‚Üí fallback MongoDB
6. Trending engine consumes Kafka ‚Üí updates ClickHouse aggregates

---

### Example 2: Live Video Chat (Omegle mode)

1. User joins ‚Üí Redis queue (`waiting_users`)
2. Matching worker pairs two users ‚Üí emits via WebSocket (Go server)
3. WebRTC handshake ‚Üí mediasoup SFU connects video streams
4. Redis keeps session state (`session:<pair>`)
5. Metrics sent to Prometheus + Kafka for analytics

---

### Example 3: Stories (auto-delete)

1. User uploads ‚Üí MongoDB (`stories`) with `expires_at`
2. TTL index removes after 24h automatically
3. Redis caches current stories for fast display
